## Scope
You should:
- improve the site structure and styling
- keep assets organized
- align implementation with the content and design artifacts
- capture meaningful decisions in governance

You should not:
- add a backend
- add a framework unless explicitly justified
- add build tooling unless the artifacts call for it
- leave major design or content decisions undocumented

## Definition of done
- The active pages are implemented or clearly scaffolded.
- Supporting artifacts match the implementation.
- The resulting structure is easy to review and extend.
