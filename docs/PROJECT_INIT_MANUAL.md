# Project Initialization Manual

1. Create a repo from this template.
2. Edit `CLAUDE.md`.
3. Fill `00_brief/`.
4. Capture the site map, page inventory, and initial copy in `01_content/`.
5. Define the visual direction and components in `02_design/`.
6. Plan build, QA, and performance checks in `03_build/`.
7. Implement the actual site in `07_site/`.
8. Keep `05_governance/` current.
9. Use `06_deploy/` only if hosting, environments, or publishing need explicit documentation.
10. For existing repos, fill `90_legacy_review/` before major changes.
